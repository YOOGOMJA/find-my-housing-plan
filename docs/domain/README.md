# Domain 문서 모음

이 폴더는 SH/LH 주택 공고 자동화에서 반복적으로 쓰는 도메인 지식을 정리합니다.

## 1) 요약

- 이 폴더의 문서는 구현 코드 설명서가 아니라, 설계와 판정 기준을 잡기 위한 도메인 기준 문서다.
- 사실 확인이 필요한 항목은 `법령/고시 원문 > 공고문 원문 > 포털 요약` 우선순위로 해석한다.
- 구현 진행 전에는 `도메인 가이드`로 개념을 고정하고, `학습 로드맵`으로 준비 상태를 점검한다.

문서 성격:

- `README.md`: 폴더 진입점
- `lh-sh-domain-guide.md`: 도메인 기준 문서
- `learning-roadmap.md`: 학습/준비 상태 점검 문서
- 공통: 구현 진행 상태를 직접 추적하는 문서는 아님

## 2) 읽는 순서

1. `lh-sh-domain-guide.md`의 `요약`과 `문서 사용 원칙`을 먼저 확인
2. `사업 vs 공고`, `청약 핵심 용어`, `LH와 SH의 차이`를 읽고 데이터 모델 기준을 잡기
3. 구현 직전에는 `수집/판정 체크리스트`를 확인해 누락 필드를 점검
4. 팀 온보딩이나 사전 학습이 필요하면 `learning-roadmap.md`로 넘어가 단계별 완료 기준 확인

## 3) 문서 목록

- [lh-sh-domain-guide.md](./lh-sh-domain-guide.md): 도메인 개념, 용어, 수집 항목, 판정 체크리스트
- [learning-roadmap.md](./learning-roadmap.md): 구현 전 학습 순서와 완료 기준

## 4) 주의

- 이 폴더의 내용에는 `운영 정의`, `권장 스키마`, `추가 검증 필요 항목`이 함께 포함된다.
- 구현 상태 자체는 별도 설계/구현 문서에서 확인하고, 이 폴더는 도메인 기준과 판단 근거 정리에 집중한다.
